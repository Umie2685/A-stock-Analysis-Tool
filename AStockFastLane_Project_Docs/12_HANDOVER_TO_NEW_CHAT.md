# 12_HANDOVER_TO_NEW_CHAT：新对话交接文档

你现在接手的是一个新项目：`AStockFastLane`。

## 项目目标

基于 `a-stock-data` 的数据源思路，快速开发一个 A 股热点分析工具。目标是自动抓取新闻、公告、研报、行情等公开信息，形成结构化 Evidence Pack，并生成 Markdown 报告。

## 工作方式

用户负责协调和运行 Codex。ChatGPT 负责整体开发节奏、Codex 任务拆分和验收。Codex 负责执行具体开发任务。

## 核心原则

- 新项目独立于旧项目。
- 以 a-stock-data 为核心参考，但不整包复制。
- 每个数据源先 probe，再 provider，再 pack，再 report。
- 不做自动交易。
- 不输出买卖建议。
- 不高频抓取。
- 不绕过鉴权或风控。

## 当前建议阶段

从 MVP0-001 开始：初始化项目骨架。

随后按顺序执行：

```text
MVP0-001：项目骨架初始化
MVP0-002：a-stock-data 参考材料审查记录
MVP0-003：东财新闻最小探针
MVP0-004：巨潮公告最小探针
MVP0-005：东财研报最小探针
MVP0-006：行情快照最小探针
MVP0-007：股票 / 主题关联规则
MVP0-008：Fast Evidence Pack
MVP0-009：Markdown 报告生成
MVP0-010：一键运行脚本
```

## 对 ChatGPT 的要求

你需要继续扮演：

- 项目经理；
- 架构把关人；
- Codex 任务拆解人；
- 验收官。

每次 Codex 交付后，先判断是否越界，再判断是否通过，最后再给下一条任务。

## 对 Codex 的要求

每次只执行一个任务。不得擅自扩大范围。不得接自动交易。不得修改未授权文件。必须运行脚本并给出输出。

## 第一条任务位置

请从 `09_CODEX_TASKS_MVP0.md` 中复制 `MVP0-001` 发给 Codex。
