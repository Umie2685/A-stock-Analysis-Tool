# 01_PROJECT_CHARTER：项目章程

## 项目名称

`AStockFastLane`

## 项目背景

A 股热点分析往往需要同时查看多个来源：新闻、公告、研报、行情、题材、行业、资金、龙虎榜等。传统手工方式效率低，且容易遗漏信息。

`a-stock-data` 提供了一个面向 AI 编程助手的 A 股数据源 Skill，公开说明中称其覆盖行情、研报、信号、资金面、新闻、基础数据、公告等层级，并采用内嵌 Python / Markdown 的方式组织端点信息。

本项目不把 `a-stock-data` 当作可直接整包复制的系统，而是把它作为数据源地图和端点参考，逐个拆出可验证、可维护、可回滚的最小功能。

## 项目目标

开发一个本地运行的 A 股热点分析工具，能够：

1. 自动抓取少量新闻、公告、研报等公开信息；
2. 关联相关股票和主题；
3. 补充行情快照、K 线或盘口摘要；
4. 生成结构化证据包；
5. 输出 Markdown 热点分析报告；
6. 保留清晰的数据质量说明和风险提示。

## 非目标

本项目不做：

- 自动交易；
- 下单接口；
- 买卖建议；
- 收益承诺；
- 高频爬虫；
- 绕过鉴权；
- 数据贩卖；
- 复杂数据库系统；
- 实盘投顾。

## 成功标准

MVP-0 成功标准：

```text
运行一个命令后，可以生成：
1. raw 数据文件；
2. cache 最新文件；
3. fast evidence pack；
4. Markdown 报告；
5. 当前进度文档。
```

## 项目角色

| 角色 | 职责 |
|---|---|
| 用户 | 协调、运行 Codex、确认方向 |
| ChatGPT | 项目经理、任务拆解、架构把关、验收 |
| Codex | 代码实现、运行测试、提交结果 |

## 开发原则

- 小步快跑。
- 每次只做一个端点或一个模块。
- 所有数据源先 probe。
- probe 成功后才封装 provider。
- provider 成功后才进入 evidence pack。
- evidence pack 成功后才进入 report。
- 出错要降级，不要中断全流程。
