# 03_SCOPE_AND_BOUNDARIES：范围与边界

## 项目范围

本项目只做公开数据的整理和分析辅助。

允许范围：

- 抓取公开新闻列表；
- 抓取公开公告列表；
- 抓取公开研报列表；
- 抓取公开行情快照或 K 线；
- 将信息结构化；
- 生成分析报告；
- 输出风险提示和待验证问题。

## 明确禁止

禁止：

- 自动交易；
- 连接券商账户；
- 生成下单指令；
- 输出“必涨”“买入”“卖出”等直接交易建议；
- 高频访问接口；
- 绕过验证码、鉴权、风控；
- 大规模转存第三方数据；
- 伪装成官方数据源；
- 隐藏数据质量问题。

## 数据使用原则

- 所有数据必须记录来源。
- 所有来源必须记录抓取时间。
- 所有接口必须设置 timeout。
- 所有失败必须写入日志或 JSON。
- 不能用单一数据源支持强结论。
- 报告必须区分：事实、市场反应、推测、待验证事项。

## 与 a-stock-data 的关系

`a-stock-data` 是本项目的数据源参考和端点地图，不是要整包嵌入的业务系统。

本项目对其使用方式：

```text
阅读 README / SKILL.md
↓
提取 endpoint 思路
↓
做本项目独立 probe
↓
封装本项目 provider
↓
写入本项目 evidence pack
```

不允许：

```text
直接复制整个 SKILL.md 到主流程
直接复制所有函数
一次性接入所有端点
跳过 probe 直接进 report
```
