# 08_CODEX_WORKFLOW：Codex 协作流程

## 协作模式

```text
用户提出目标
↓
ChatGPT 拆成小任务
↓
用户把任务发给 Codex
↓
Codex 修改代码并运行
↓
用户把结果贴回 ChatGPT
↓
ChatGPT 验收 / 返工 / 发下一任务
```

## 给 Codex 的任务必须包含

每条任务必须写清：

1. 任务编号；
2. 任务名称；
3. 背景；
4. 目标；
5. 允许修改文件；
6. 禁止修改文件；
7. 运行方式；
8. 输出文件；
9. 交付格式；
10. 验收标准。

## Codex 禁止行为

- 一次性重构整个项目；
- 擅自接入 LLM；
- 擅自接入数据库；
- 擅自做并发爬取；
- 擅自接入 i问财；
- 擅自修改主 evidence 结构；
- 擅自删除现有文档；
- 不运行测试就声称完成。

## 验收维度

ChatGPT 每次验收看：

- 文件是否按要求新增 / 修改；
- 是否越界；
- 是否可运行；
- 是否有输出文件；
- 是否有异常处理；
- 是否有文档；
- 是否保持小步迭代；
- 是否适合进入下一步。

## Codex 交付标准格式

```text
1. 修改了哪些文件
2. 如何运行
3. 实现了哪些功能
4. 哪些成功
5. 哪些失败
6. 生成了哪些文件
7. 是否修改了禁止修改的文件
8. 是否有未解决问题
9. 下一步建议
```

## 返工原则

如果 Codex 越界，优先回滚越界部分；如果接口失败，优先保留错误记录和文档，不强行写死成功。
