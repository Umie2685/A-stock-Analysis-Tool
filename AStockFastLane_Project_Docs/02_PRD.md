# 02_PRD：产品需求文档

## 产品名称

AStockFastLane：A 股热点信息快速分析工具

## 目标用户

- 个人投资研究者；
- 需要快速汇总 A 股公开信息的开发者；
- 想把 AI / Codex 用到投研资料整理中的用户。

## 核心用户故事

### 用户故事 1：自动抓取当天新闻

作为用户，我希望工具能够自动抓取少量市场新闻，并保存为结构化 JSON，这样我不用手工复制新闻标题。

验收标准：

- 能抓取新闻标题、发布时间、来源、链接、摘要或正文片段；
- 单次抓取数量可控，例如 20 条以内；
- 失败时有错误记录。

### 用户故事 2：自动抓取公告

作为用户，我希望工具能够抓取指定股票或关键词相关公告，尤其是巨潮公告，这样可以把信息披露纳入分析证据。

验收标准：

- 能按股票代码或关键词抓取公告列表；
- 能记录公告标题、时间、公司、公告链接；
- 不强制下载 PDF 正文。

### 用户故事 3：自动抓取研报

作为用户，我希望工具能够抓取少量研报摘要或列表，这样可以了解机构观点变化。

验收标准：

- 能抓取研报标题、机构、发布时间、相关股票、链接；
- 不把研报观点当作事实；
- 记录来源和数据质量限制。

### 用户故事 4：形成 Evidence Pack

作为用户，我希望所有抓到的信息能够被整理成一个结构化证据包，方便后续 AI 生成报告。

验收标准：

- 输出 `data/evidence/fast_evidence_pack_latest.json`；
- 包含新闻、公告、研报、行情摘要；
- 包含数据质量说明。

### 用户故事 5：生成 Markdown 报告

作为用户，我希望工具可以输出一份可读报告，展示热点、相关股票、证据、风险和待验证问题。

验收标准：

- 输出 `reports/fastlane_report_YYYYMMDD.md`；
- 报告中明确区分事实、推测、待验证事项；
- 不输出买入 / 卖出建议。

## MVP-0 功能清单

| 功能 | MVP-0 是否实现 | 说明 |
|---|---:|---|
| 东财新闻探针 | 是 | 首个数据源 |
| 巨潮公告探针 | 是 | 信息披露核心 |
| 东财研报探针 | 是 | 机构观点补充 |
| 行情快照 | 是 | 可用 Tencent 或 mootdx |
| 主题 / 股票关联 | 简化实现 | 规则 + manual mapping |
| Evidence Pack | 是 | JSON |
| Markdown 报告 | 是 | 不接 LLM 也可先生成模板报告 |
| LLM 分析 | 暂缓 | 后续接 DeepSeek / OpenAI |
| Streamlit 页面 | 暂缓 | 先 CLI |
| 数据库 | 暂缓 | 先文件缓存 |
| 自动交易 | 永不实现 | 项目边界 |

## MVP-1 功能候选

- LLM 报告润色；
- 主题归因；
- 股票池 watchlist；
- 历史缓存；
- 多数据源交叉验证；
- 简单 Web UI；
- 定时任务；
- 失败重试与限流策略。
