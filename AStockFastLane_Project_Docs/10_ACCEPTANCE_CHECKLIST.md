# 10_ACCEPTANCE_CHECKLIST：验收清单

## 每个 Codex 任务通用验收

| 检查项 | 是否必须 |
|---|---:|
| 修改文件符合允许范围 | 是 |
| 未修改禁止文件 | 是 |
| 脚本可以运行 | 是 |
| 失败有受控处理 | 是 |
| 输出 raw/cache 文件 | 视任务 |
| 文档已更新 | 是 |
| current_progress 已更新 | 是 |
| 没有自动交易逻辑 | 是 |
| 没有高频请求 | 是 |
| 没有硬编码不可解释魔法参数 | 是 |

## Probe 任务验收

- 是否记录 endpoint；
- 是否记录请求参数；
- 是否设置 timeout；
- 是否限制请求数量；
- 是否保存 sample 数据；
- 是否记录字段名；
- 是否记录错误类型；
- 是否记录空返回；
- 是否写 probe 文档。

## Provider 任务验收

- 是否有输入校验；
- 是否返回结构化 dict；
- 是否不返回裸 response；
- 是否包含 ok/status/error；
- 是否没有直接写报告；
- 是否没有强依赖特定日期；
- 是否没有过度抽象。

## Evidence Pack 任务验收

- 是否只读取 cache；
- 是否不请求外网；
- 是否包含 sources；
- 是否包含 data_quality；
- 是否保留 unmatched 项；
- 是否结构适合报告生成；
- 是否不塞入过大 raw 数据。

## Report 任务验收

- 是否只读取 Evidence Pack；
- 是否不请求外网；
- 是否不输出投资建议；
- 是否包含数据质量说明；
- 是否区分事实 / 推测 / 待验证；
- 是否包含来源列表。

## 一票否决项

出现以下行为直接判定不通过：

- 接入自动交易；
- 删除主数据文件；
- 大规模并发爬取；
- 绕过鉴权；
- 把第三方代码整包复制进项目；
- 把报告写成买卖建议；
- 不运行脚本就声称完成；
- 修改了明确禁止修改的文件。
